Para tornar o Mathemagician mais viciante e recompensador para crianças do 1.º ao 4.º ano, o foco deve estar na progressão visual e em ciclos de feedback curto.

Aqui estão sugestões estruturadas para elevar a retenção:

1. Sistema de Colecionáveis e Personalização
Crianças nesta faixa etária respondem bem à posse e identidade visual.

Avatar Evolutivo: O utilizador começa como um "Aprendiz de Feiticeiro". Ao completar reinos, ganha itens cosméticos (chapéus, varinhas, capas).

Bestiário/Grimório: Cada nível completado pode libertar uma "criatura mágica" que fica guardada num álbum. O objetivo passa a ser "completar a coleção".

2. Feedback Imediato e "Game Feel"
O splash screen atual é um bom começo, mas pode ser mais dinâmico:

Combo Strikes: Se a criança acertar 3 questões seguidas, a interface ganha efeitos visuais (faíscas, mudança de cor).

Barras de Experiência (XP): Em vez de apenas níveis, mostre uma barra que enche a cada resposta certa. O cérebro humano reage positivamente a barras de progresso a encher.

3. Moeda Virtual e Economia Simples
Cristais de Mana: Cada resposta correta dá 1 a 5 cristais.

Loja Mágica: Os cristais não compram vantagens no jogo (para não estragar a aprendizagem), mas compram cores novas para o portal ou efeitos de som diferentes para quando acertam.

4. Estrutura de Narrativa e Desafio
Elemento	Implementação
Boss Battle	A 5.ª questão do último nível de cada ano pode ter uma animação diferente (um dragão ou um feiticeiro rival) que é "derrotado" com a resposta certa.
Daily Quest	"Resolve 5 subtrações hoje para ganhar um bónus de XP." Isto incentiva o uso diário da app.
Streak (Série)	Mostrar há quantos dias seguidos a criança estuda. Manter a "chama mágica" acesa.
5. Mecânicas de Retenção (Push)
Certificados Mágicos: Ao completar um ano (4 níveis), a app gera um PDF ou imagem de "Mestre da Aritmética" que a criança pode mostrar aos pais.

Sistemas de Ranking (Opcional): Se não quiser competitividade direta, use "Ligas Mágicas" baseadas em pontos totais, onde o utilizador sobe de categoria (Bronze, Prata, Ouro).

Sugestão Técnica de Melhoria
Dado que a app está em Vercel (provavelmente Next.js ou similar), pode adicionar micro-interações usando bibliotecas como framer-motion para o splash screen ou canvas-confetti para o momento em que terminam o reino.

Para garantir que a implementação seja precisa, estruturei as User Stories focadas em funcionalidade técnica, separando a lógica de progressão da interface visual.

Epic: Gamification e Progressão Narrativa
US1: Sistema de Inventário e Persistência de Itens
Como sistema da aplicação,
Quero registar e armazenar os itens mágicos conquistados pelo utilizador,
Para que o progresso seja guardado entre sessões.

Critérios de Aceitação:

Criar um objeto de estado (ex: userProgress) que armazene: currentRealm, completedLevels e unlockedItems (array).

Utilizar localStorage (ou base de dados) para persistir estes dados.

Implementar uma função unlockItem(itemId) que adiciona o item ao array apenas se ele ainda não existir.

Definição de Pronto: O utilizador pode fechar o browser, voltar, e os itens ganhos continuam visíveis no seu perfil/inventário.

US2: Splash Screens Dinâmicos por Conquista
Como utilizador,
Quero ver mensagens e visualizações diferentes dependendo do meu progresso (Nível vs. Reino vs. Ano),
Para que sinta uma recompensa proporcional ao esforço.

Critérios de Aceitação:

Trigger Nível Intermédio: Exibir componente Modal com animação simples de confetis e texto "Nível Concluído!".

Trigger Fim de Reino (Meio Ano): Exibir Modal ocupando 80% do ecrã com a imagem do Item Mágico ganho (ex: Cajado) e o texto: "Conquistaste o [Nome do Reino]! Recebeste o [Item]. Estás a meio do [Ano]º Ano!".

Trigger Fim de Ano (4 Níveis): Exibir Splash Screen em ecrã total com animação dourada e texto: "Mestre do [Ano]º Ano! O teu poder cresceu.".

Definição de Pronto: O sistema identifica o ID do nível/reino e injeta o conteúdo correto no componente de Splash Screen.

US3: Visualização do Inventário (O Baú Mágico)
Como utilizador,
Quero aceder a uma secção onde vejo todos os itens que já ganhei,
Para que possa visualizar o meu histórico de conquistas.

Critérios de Aceitação:

Criar uma rota ou componente /inventory (ou ícone de baú na home).

Apresentar uma grelha (grid) com 8 slots (um para cada item dos reinos).

Itens não conquistados devem aparecer em cinzento/silhueta (lock state).

Itens conquistados devem aparecer a cores e com o nome visível.

Definição de Pronto: A grelha de itens reflete exatamente o estado do userProgress.

US4: Feedback Visual de "Combo" (Game Feel)
Como utilizador,
Quero ver efeitos visuais quando acerto várias perguntas seguidas,
Para que a resolução de problemas seja mais estimulante.

Critérios de Aceitação:

Implementar um contador streakCount que faz reset ao errar uma pergunta.

Se streakCount >= 3, aplicar uma classe CSS de animação (ex: glow-effect) ao contentor da pergunta.

Ao acertar com streak, disparar componente de partículas (ex: canvas-confetti) com cores temáticas do reino atual.

Definição de Pronto: O feedback visual intensifica-se conforme a sequência de acertos do utilizador.

## Reacnt/Next.js suggestion
Aqui tens uma estrutura limpa e modular em React/Next.js para gerires o progresso e as recompensas:

1. O Hook de Estado (useGamification.js)
Este ficheiro centraliza a lógica de guardar no localStorage e verificar se um item foi desbloqueado.

JavaScript
import { useState, useEffect } from 'react';

export const useGamification = () => {
  const [progress, setProgress] = useState({
    completedRealms: [],
    unlockedItems: [],
    currentStreak: 0,
  });

  // Carregar progresso ao iniciar
  useEffect(() => {
    const saved = localStorage.getItem('mathemagician_progress');
    if (saved) setProgress(JSON.parse(saved));
  }, []);

  const completeLevel = (realmId, isEndOfRealm, isEndOfYear) => {
    let newItems = [...progress.unlockedItems];
    let newRealms = [...progress.completedRealms];

    if (isEndOfRealm && !newRealms.includes(realmId)) {
      newRealms.push(realmId);
      // Lógica de mapeamento de itens (exemplo simplificado)
      const itemMap = { 'R1': 'Cajado', 'R2': 'Manto' };
      if (itemMap[realmId]) newItems.push(itemMap[realmId]);
    }

    const newState = {
      ...progress,
      completedRealms: newRealms,
      unlockedItems: newItems,
      currentStreak: progress.currentStreak + 1
    };

    setProgress(newState);
    localStorage.setItem('mathemagician_progress', JSON.stringify(newState));
    return { isEndOfRealm, isEndOfYear, itemGained: itemMap[realmId] };
  };

  return { progress, completeLevel };
};

export const RewardModal = ({ isOpen, type, itemName, onClose }) => {
  if (!isOpen) return null;

  return (
    <div className="fixed inset-0 bg-black/50 flex items-center justify-center z-50">
      <div className="bg-white p-8 rounded-xl text-center max-w-sm animate-bounce-in">
        <h2 className="text-2xl font-bold mb-4">
          {type === 'year' ? '🏆 Mestre do Ano!' : '✨ Reino Conquistado!'}
        </h2>
        
        {itemName && (
          <div className="my-6">
            <div className="text-6xl mb-2">🎁</div>
            <p className="font-medium text-purple-600">Ganhaste: {itemName}</p>
          </div>
        )}

        <p className="text-gray-600 mb-6">
          {type === 'year' 
            ? 'Completaste todos os desafios deste ano escolar!' 
            : 'Estás a meio caminho do teu objetivo mágico.'}
        </p>
        
        <button 
          onClick={onClose}
          className="bg-purple-600 text-white px-6 py-2 rounded-full hover:bg-purple-700 transition"
        >
          Continuar Jornada
        </button>
      </div>
    </div>
  );
};
